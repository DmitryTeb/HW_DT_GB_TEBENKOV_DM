# Credit Default Prediction

## Описание проекта

Цель проекта — построить модель машинного обучения для прогнозирования вероятности невыполнения клиентом кредитных обязательств (Credit Default) на основе данных о финансовом состоянии заемщиков.

В рамках работы была реализована модель классификации, обученная на исторических данных клиентов банка. Модель используется для предсказания дефолта на тестовом наборе данных.

Основной метрикой оценки качества является **F1-score для класса 1 (кредитный дефолт)**.

---

# Данные

Используются два набора данных:

- `course_project_train.csv` — обучающая выборка  
- `course_project_test.csv` — тестовая выборка  

### Размеры датасетов

| Dataset | Размер |
|-------|-------|
| Train | 7500 × 17 |
| Test | 2500 × 16 |

### Целевая переменная

`Credit Default`

Значения:

- **0** — клиент выполнил кредитные обязательства
- **1** — клиент допустил просрочку по кредиту

### Распределение классов

| Класс | Количество | Доля |
|------|------|------|
| 0 | 5387 | 71.8% |
| 1 | 2113 | 28.2% |

Наблюдается дисбаланс классов, поэтому при обучении моделей используется балансировка.

---

# Разведочный анализ данных (EDA)

На этапе EDA были выполнены:

- анализ структуры данных
- анализ пропусков
- исследование распределений признаков
- анализ выбросов
- построение корреляционной матрицы
- анализ влияния признаков на вероятность дефолта

### Пропуски в данных

| Признак | Доля пропусков |
|------|------|
| Months since last delinquent | 54% |
| Credit Score | 21% |
| Annual Income | 21% |
| Years in current job | 5% |

Пропуски обрабатываются следующим образом:

- числовые признаки — заполнение медианой
- категориальные признаки — заполнение наиболее частым значением

# Feature Engineering

Для улучшения качества модели были созданы дополнительные признаки.

### Debt-to-Income (DTI)

Показатель долговой нагрузки клиента.

Формула:

Gini = 2 * AUC - 1

# Результаты моделей

| Модель | F1 | AUC | Gini |
|------|------|------|------|
| Logistic Regression | 0.522 | 0.754 | 0.508 |
| Decision Tree | 0.432 | 0.731 | 0.463 |
| Random Forest | **0.541** | 0.765 | 0.529 |
| Gradient Boosting | 0.440 | 0.760 | 0.520 |

Лучший результат показала модель **Random Forest**.

---

# Финальная модель

В качестве финальной модели выбран **Random Forest**.

Параметры модели:

n_estimators = 400
max_depth = 12
min_samples_leaf = 10
class_weight = "balanced"

Результат на валидационной выборке:

F1-score = 0.541

# Итог

В рамках проекта был реализован полный pipeline разработки ML-модели:

1. Проведен разведочный анализ данных (EDA)
2. Выполнена обработка пропусков
3. Проанализированы выбросы
4. Созданы дополнительные признаки
5. Реализован Pipeline предобработки
6. Обучены и сравнены несколько моделей машинного обучения
7. Выбрана финальная модель

Лучшая модель — **Random Forest**, показавшая наилучший результат по метрике F1-score.

Финальная модель обучена на всей обучающей выборке и использована для построения прогнозов на тестовом датасете.
